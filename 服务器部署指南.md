## 🖥️ 准备服务器

### 方式一：自动初始化（推荐）

在你的服务器上运行：

```bash
# 下载并运行初始化脚本
curl -O https://raw.githubusercontent.com/你的用户名/devops-actions/main/scripts/server-setup.sh
chmod +x server-setup.sh
sudo ./server-setup.sh
```

### 方式二：手动安装

```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 2. 启动 Docker
sudo systemctl enable docker
sudo systemctl start docker

# 3. 创建部署目录
sudo mkdir -p /var/www/devops-actions
```

## 🔐 配置 GitHub Secrets

在 GitHub 仓库中进入：
**Settings → Secrets and variables → Actions → New repository secret

添加以下 Secrets：

| Secret 名称 | 说明 | 示例 |
|-------------|------|------|
| `SSH_HOST` | 服务器 IP 地址 | `123.45.67.89` |
| `SSH_USER` | 服务器用户名 | `root` 或 `ubuntu` |
| `SSH_PASS` | 服务器密码 | `your-password` |
| `SSH_PORT` | SSH 端口（可选） | `22` |
| `DEPLOY_PATH` | 部署路径（可选） | `/var/www/devops-actions` |
| `GHCR_PAT` | GitHub PAT（拉取镜像用） | `ghp_xxxxxxxxxxxx` |

### 如何创建 GitHub PAT (Personal Access Token)

1. 进入 GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 点击 Generate new token (classic)
3. 勾选这些权限：
   - `write:packages`
   - `read:packages`
   - `repo` (如果是私有仓库)
4. 生成后复制，保存到 `GHCR_PAT` Secret

## 🚀 开始部署

### 方式一：通过 GitHub Actions 界面

1. 进入你的 GitHub 仓库
2. 点击 **Actions** 标签
3. 选择 **CD Pipeline** 工作流
4. 点击 **Run workflow**
5. 选择分支：`main`
6. 选择部署类型：`ssh`
7. 点击 **Run workflow**

### 方式二：通过打 Tag 触发

```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

然后在 Actions 中手动选择 `ssh` 部署。

## ✅ 验证部署

部署完成后，验证：

```bash
# 在服务器上检查容器
docker ps

# 查看日志
docker logs devops-actions

# 测试访问
curl http://localhost
```

在浏览器访问：`http://你的服务器IP`

## 🔄 回滚

如果需要回滚：

```bash
# 停止当前容器
docker stop devops-actions

# 启动旧版本（假设你有备份）
docker run -d --name devops-actions --restart=always -p 80:80 ghcr.io/你的用户名/devops-actions:旧版本号
```

## 📊 本地 SSH 部署测试

在本地也可以直接 SSH 部署测试：

```bash
# 复制文件到服务器
scp -r app/* root@你的服务器IP:/var/www/devops-actions/

# 或者用 Docker
ssh root@你的服务器IP "docker pull ghcr.io/你的用户名/devops-actions:latest"
```

---

## 💡 提示

- 如果使用 SSH 连接失败，检查：
  1. 服务器防火墙是否放行了 22 端口
  2. 用户名密码是否正确
  3. 服务器是否可以访问 GitHub Container Registry
