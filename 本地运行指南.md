# 本地运行指南

## 🚀 方式一：直接打开（推荐，最简单）

1. 打开文件夹：`e:\github\devops-actions\app`
2. 双击 `index.html` 文件
3. 浏览器会自动打开展示页面

---

## 🚀 方式二：用 Node.js 启动服务器

如果你有 Node.js：

```bash
cd .github/actions/gray-release
npx serve -p 8080 ../../app
```

然后访问：http://localhost:8080

---

## 🚀 方式三：用 Docker 运行

如果你有 Docker：

```bash
# 构建镜像
docker build -t devops-actions .

# 运行容器
docker run -d -p 8080:80 --name devops-demo devops-actions

# 访问
# 浏览器打开: http://localhost:8080

# 停止和删除
docker stop devops-demo
docker rm devops-demo
```

---

## 📱 测试 GitHub Action 本地

虽然 GitHub Actions 主要在 GitHub 上运行，但你可以测试 Action 的代码：

```bash
cd .github/actions/gray-release

# 安装依赖（如果还没安装）
npm install

# 构建
npm run build

# 测试运行（需要 mock 环境）
# 或者直接看 src/index.js 了解逻辑
```

## 🎯 项目展示要点

向面试官展示时，重点讲：

1. **前端应用** - 演示界面，点击"模拟部署"和"灰度切换"
2. **CI/CD 流水线** - 打开 `.github/workflows/` 下的 yml 文件讲解
3. **自定义 Action** - 展示 `.github/actions/gray-release/` 的代码
4. **Docker 支持** - 展示 Dockerfile 和 nginx.conf
